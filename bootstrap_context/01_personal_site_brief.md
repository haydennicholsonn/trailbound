# 01 — Personal Site Brief

## Project

**haydenn.co.za**

## Public-facing name

**hayden**

## One-line identity

> dev who enjoys games and building cool things

Alternative polished version:

> I build useful systems, game-like web experiences, and weirdly polished things for the web.

## Goal

Create a personal showcase site that feels like a cinematic, interactive developer/game HUD experience.

This is not a recruitment CV page. It is not a corporate profile. It is a personal digital space that makes people think:

> This is polished. This person can build cool shit.

The site should introduce Hayden, show his creative/developer taste, and focus attention on one flagship project: the gamified running app.

## Target audience

Primary:

- Friends
- People who are curious what Hayden builds
- People who might later use the running app
- Developers or technical people who appreciate polish

Secondary:

- Anyone landing on haydenn.co.za who wants to understand what Hayden does

Not the target:

- Recruiters as the main audience
- Corporate clients
- A formal CV audience

## Tone

Blend professional and personality-driven.

The site should sound confident, direct, and human.

Use copy like:

> I build practical web systems, interactive experiences, and game-inspired tools.

But it can also have personality:

> I like taking boring flows and making them feel better, faster, and more fun.

Avoid:

- Generic corporate portfolio language
- Buzzword stuffing
- Fake startup hype
- Mentioning employers or company names
- Overexplaining everything

## Brand direction

The visual style should be:

**Cyberpunk × Apple premium × game HUD**

This means:

- Dark cinematic background
- Sharp typography
- Premium spacing
- Glossy panels
- Subtle neon/rune lighting
- HUD-style overlays
- Game-like interactions
- Minimal clutter
- High polish
- Effects used intentionally, not everywhere

## Site structure

The site should be one interactive scrolling page where each scroll position feels like a new scene.

Not a traditional page with boring stacked sections.

Think:

```text
Scene 1: Boot / Hero
Scene 2: Identity / Stats
Scene 3: System / Skills
Scene 4: Featured Project
Scene 5: Running App Deep Dive
Scene 6: Final Contact / Continue
```

The scroll should feel like moving through a cinematic interface.

## Main sections

### 1. Boot / Hero scene

Purpose:

Introduce the site and immediately set the tone.

Suggested copy:

```text
hayden
dev who enjoys games and building cool things

I build practical web systems, interactive experiences, and game-inspired tools.
Currently building a fantasy RPG running app that turns real-world runs into quests, loot, XP, and progression.
```

Primary button:

```text
Enter the showcase
```

Secondary button:

```text
View the project
```

Interaction ideas:

- On load, fake boot sequence: `initialising haydenn.co.za`
- HUD grid fades in
- Logo/avatar appears from light streaks
- Mouse movement creates very subtle parallax
- Button hover emits small particles
- Button click shatters into clean pixel/cube blocks before moving to next scene

### 2. Identity / player card

Purpose:

Show Hayden as a “player profile” rather than a CV.

Suggested UI:

```text
PLAYER PROFILE
Name: hayden
Class: Developer / Builder
Traits: Practical, curious, systems-minded, game-brained
Focus: Web apps, APIs, dashboards, tools, game-like experiences
```

Suggested copy:

```text
I like building things that feel good to use. Clean systems, useful dashboards, smooth flows, and projects with enough polish to make them memorable.
```

### 3. Skills / system stats

Purpose:

Show skills in a game HUD way without feeling like LinkedIn.

Suggested stat cards:

```text
Backend
PHP / MySQL / APIs / Auth / Cron

Frontend
JavaScript / UI flows / Dashboards / Maps

Systems
Data design / Reporting / Automation / Integrations

Creative
Game systems / Progression / Interactive UI / Polish
```

Alternative style:

```text
SKILL TREE UNLOCKED
- PHP backend
- MySQL data modelling
- API integrations
- Interactive dashboards
- Map-based systems
- Game progression design
```

### 4. Featured project teaser

Purpose:

Introduce the gamified running app as the only showcased project.

Working title:

**Project Trailbound**

Suggested copy:

```text
Featured build

Project Trailbound is a fantasy RPG running app where your real runs unlock quests, XP, loot, areas, gear, and character progression.

You run in the real world.
Your character progresses in the game world.
```

Primary button:

```text
Open project dossier
```

### 5. Project deep dive

Purpose:

Make the running app feel exciting and real.

Sections inside the scene:

- The problem: running can feel repetitive
- The idea: make every run feed progression
- The loop: Run -> Sync -> Rewards -> Unlock -> Plan next quest
- The fantasy: adventure map, loot, skill tree, avatar, quests
- The tech: Strava connection, PHP/MySQL backend, Mapbox map, gamified frontend

Suggested copy:

```text
The goal is simple: make running feel like progression.

Complete a 3km run and unlock a forest path.
Hit a weekly streak and open a reward chest.
Explore a new route and reveal part of the world map.
Level up and spend points in a passive tree.
```

### 6. Final scene / contact

Purpose:

End cleanly.

Suggested copy:

```text
This site is my little corner of the internet.
A place for experiments, polished ideas, and things that probably started with “what if this was more game-like?”
```

Button ideas:

```text
Follow the build
Contact
Restart sequence
```

No CV button for now.

## Content rules

- Do not mention any employer.
- Do not mention Ikeja.
- Do not list work projects yet.
- Do not make the site feel job-hunty.
- Do not use Hayden’s face.
- Do not create a fake long portfolio.
- Keep the running app as the flagship focus.
- Keep all copy short, punchy, and intentional.

## Success criteria

The site is successful when:

- It feels custom, not template-based.
- It feels premium, not gimmicky.
- It clearly explains who Hayden is.
- It clearly points to the running app.
- It makes friends/devs say the site is cool.
- It runs smoothly on desktop and mobile.
- It can be hosted on Afrihost/cPanel without complex server dependencies.
